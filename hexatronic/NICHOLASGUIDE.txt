12) MAKE APP PRIVATE WITH ENTRA ID (NO PUBLIC ACCESS)
============================================================

Goal: users must sign in with Entra ID before using the site.

A) Create a separate Entra App Registration for USER LOGIN
   (Do NOT use the GitHub OIDC deployment app)
   - Name example: supplier-compare-web-login
   - Single-tenant
   - Redirect URI (Web):
     https://<app-url>/.auth/login/aad/callback
   - Create client secret and store its value in Azure Key Vault
     (secret name: entra-login-client-secret, or your custom name)

B) Configure Container App Authentication
   - Azure Portal -> Container App (<ENVIRONMENT_NAME>-app) -> Authentication
   - Add identity provider: Microsoft
   - Use the login app registration client ID and secret
   - Issuer/tenant: your tenant (single-tenant)
   - Set unauthenticated requests to:
     "Require authentication" (or Return 401)

C) Important for health probe
   - Exclude path /health from auth (allow anonymous),
     otherwise liveness checks can fail.

D) Restrict who can access (optional but recommended)
   - Entra ID -> Enterprise Applications -> your login app
   - Assign specific users/groups only.

Result:
- App is no longer publicly usable.
- Users must authenticate with Entra ID.
